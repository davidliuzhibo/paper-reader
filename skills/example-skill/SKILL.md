## Overview

This is an example skill that shows the basic structure of a SKILL.md file.

## When to Use

This skill activates when:
- User asks about skill structure
- User wants to see an example skill
- User needs a template for creating skills

## Instructions

When this skill is triggered:

1. Explain what the skill does
2. Show the user the basic structure
3. Provide guidance on customization

## Key Components

### Name
- Use lowercase with hyphens
- Keep it descriptive and concise
- Example: `pdf-editor`, `brand-guidelines`

### Description
- Most critical for triggering
- Write from Claude's perspective
- Include specific capabilities and use cases
- Define clear boundaries

### Instructions
- Use markdown formatting
- Break into clear sections
- Include examples
- Specify limitations

## Example Usage

When a user asks "How do I create a skill?", this skill would activate and provide guidance on the structure and components needed.
